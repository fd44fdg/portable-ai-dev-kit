## Root cause

The target `F:` removable drive (NTFS, USB 2.0-class, ~15 GB total) was
measured at **233 ms per small-file write** and **0.5 MB/s sequential**. The
npm install path (`install_npm_tool` in `portable.rs`) writes directly onto
the slow drive with no local SSD staging, no atomic swap, and no rollback.
Large npm packages (Claude Code's `node_modules` = 444 MB / thousands of
files) inevitably exceed the 10-minute `NPM_INSTALL_TIMEOUT`, leaving the
tool in a half-written "partial" state.
